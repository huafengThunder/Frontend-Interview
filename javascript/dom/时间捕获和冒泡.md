#### 事件捕获/冒泡
- 事件捕获
  - 事件发生的顺序:从根元素触发到目标元素
- 事件冒泡
  - 事件发生的顺序:从目标元素到根元素
- w3c提出方案
  - 同时支持事件捕获和冒泡
- 事件代理
  - 通过给父级元素绑定事件,在通过捕获或者冒泡事件流触发
- 阻止捕获/冒泡
  - 通过事件源直接阻止
  - return false
  - stopPropagation()
  - target==currentTarget;当触发事件的元素(事件源/元素本身)等于当前活动事件的元素(随便事件流变动的元素)。
- 阻止默认事件
  - event.prventDefault()
  - return false
